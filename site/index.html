Agent-native UI runtime

One app. Read by humans, derived by AI.

QORM apps are small, language-neutral JSON — rendered live, signed, delivered over-the-air, and packaged for web, iOS, Android, and desktop from one pure-Go runtime. Humans install and edit them; AI agents discover and verify them.

Dual-consumer by design

The same artifact serves both sides

Most tools pick an audience. QORM is built so one app is equally usable by the person shipping it and the agent reasoning about it.

For human developers

  • Readable JSON — no build step to understand an app.
  • Installs as a real app: web, iOS, Android, desktop.
  • Extend it in one Go file — the same code runs everywhere.

For AI agents

  • Discoverable: a machine-readable capability + widget catalog.
  • Drivable over MCP — read, edit, and preview a live app.
  • Self-verifiable: measure the render, check edits against truth.
Capabilities

A whole runtime, not a template

One JSON, every platform

The same app renders in a browser, snapshots to static HTML, and packages as web, iOS, Android, and desktop.

Verify-the-bundle delivery

Apps compile to a signed, content-addressed bundle. The runtime checks integrity and authenticity before running a line — safe over-the-air with rollback.

40+ native capabilities

Camera, location, Bluetooth, NFC, biometrics, sensors, safe-area insets — a single bridge contract, one machine-readable registry.

Middle-layer in one Go file

Add your own native ops in native/desktop.go — compiled into the desktop binary and the mobile/web WASM alike.

Localized out of the box

ICU messages — plurals, gender, currency, dates, and full RTL — resolved by the runtime, not bolted on.

Self-verifying for agents

qorm measure and qorm check render the app and report the real geometry, so an agent can grade its own edits against ground truth.

Extend it — recommended

Your own logic, everywhere, in Go

Register custom operations in one file. Reach the hardware bridge with qormext.Native, push events to the UI with qormext.Emit, and return a line of JS to update the app. It compiles into the desktop binary and the mobile/web WASM — one Go file, every target.

native/desktop.go
func init() {
  qormext.Register("hash", func(d map[string]any) string {
    sum := sha256.Sum256([]byte(d["text"].(string)))
    return "qormOnHash(" + quote(hex(sum)) + ")"
  })
  qormext.Native("vibrate", `{"ms":200}`)  // framework hardware
  qormext.Emit("orderDone", `{"id":42}`)  // push to the UI bus
}
Get started

From JSON to a signed app

Pure Go, no cgo — it cross-compiles to every platform from any machine.

01

Run

qorm run examples/counter
Opens the app live in your browser.

02

Sign

qorm build app -o app.bundle --key k
One signed, verifiable artifact.

03

Package

qorm package app -p ios
Web, iOS, Android, or desktop.

04

Verify

qorm check app --audit
Render, measure, grade against truth.
